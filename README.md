# PRD Agent

Telegram-бот для продуктовых менеджеров. Помогает писать PRD: задаёт уточняющие вопросы, генерирует структурированный документ и сохраняет его в базу знаний для последующего поиска.

## Стек

- **LangGraph** — граф состояний агента (приём → вопросы → генерация → сохранение)
- **ChromaDB** — векторное хранилище для RAG
- **OpenAI API** — генерация и эмбеддинги
- **aiogram 3** — Telegram-бот
- **Python 3.14**

## Структура проекта

```
prd_agent/
├── config/
│   ├── prd_template.md       # Шаблон PRD (9 секций)
│   └── models_pricing.json   # Таблица цен OpenAI моделей
├── data/
│   ├── prd_synthetic/        # Синтетические PRD для RAG (генерируются скриптом)
│   └── prd_output/           # PRD, созданные пользователем через бота
├── chroma_db/                # Векторная база ChromaDB (создаётся при индексации)
├── tests/
├── generate_synthetic.py     # Шаг 1: генерация синтетической базы PRD
├── search.py                 # Шаг 2: ChromaDB — индексация и поиск по PRD
├── agent.py                  # Шаг 3: LangGraph агент
├── bot.py                    # Шаг 5: Telegram-бот
├── requirements.txt
└── .env.example
```

## Быстрый старт

```bash
# 1. Активировать venv (находится в родительском каталоге)
source ../.venv/bin/activate

# 2. Скопировать и заполнить переменные окружения

# Открыть .env и вставить OPENAI_API_KEY и TELEGRAM_BOT_TOKEN

# 3. Сгенерировать синтетическую базу PRD
python generate_synthetic.py --count 50
```

## Шаблон PRD

Единый шаблон для всех документов (`config/prd_template.md`):

| Секция | Что содержит |
|---|---|
| Проблема | Боль пользователя или бизнеса |
| Цель | Измеримый результат |
| Пользователи | Целевая аудитория и контекст |
| Пользовательские сценарии | User stories (Как / хочу / чтобы) |
| Функциональные требования | Что система должна уметь |
| Вне скоупа | Что явно не входит в эту версию |
| Метрики успеха | Показатели, по которым оценим результат |
| Открытые вопросы | Нерешённые вопросы |
| Зависимости и риски | Зависимости от других команд и систем |

## Поиск по базе PRD

Поиск использует ChromaDB с эмбеддингами OpenAI (`text-embedding-3-small`).

```bash
# Индексировать синтетические PRD (нужно сделать один раз после генерации)
python search.py --index

# Индексировать другую папку (например, реальные PRD от пользователя)
python search.py --index --dir data/prd_output

# Найти похожие PRD по запросу
python search.py "как реализовать онбординг нового пользователя"

# Изменить количество результатов
python search.py "программа лояльности" --top 3
```

Результаты выводятся с оценкой релевантности (0–1) и превью документа.

## Генерация синтетических PRD

Синтетическая база нужна для наполнения RAG — без неё агент не может опираться на примеры прошлых документов при генерации нового PRD.

```bash
# Сгенерировать 50 PRD по всем доменам (рекомендуется)
python generate_synthetic.py --count 50

# Только один домен
python generate_synthetic.py --count 20 --domain fintech

# Другая модель
python generate_synthetic.py --count 50 --model gpt-4o
```

**Доступные домены:** e-commerce, fintech, edtech, healthtech, b2b saas, marketplace, social, productivity, logistics, hr-tech

Файлы сохраняются в `data/prd_synthetic/` в формате Markdown с именами вида `001_fintech_аналитика_расходов.md`.

## Флоу агента (в разработке)

```
PM пишет в Telegram описание фичи
    │
    ▼
Агент ищет похожие PRD в RAG (ChromaDB)
    │
    ▼
Агент задаёт 3–5 уточняющих вопросов
    │
    ├── PM отвечает → агент учитывает ответы
    │
    └── PM пишет /skip → PRD создаётся с пометкой "требует доработки"
    │
    ▼
Агент генерирует PRD по шаблону
    │
    ▼
Отправляет .md файл в Telegram + сохраняет в RAG
```
