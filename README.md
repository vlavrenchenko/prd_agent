# PRD Agent

Telegram-бот для продуктовых менеджеров. Генерирует PRD по описанию фичи, оценивает качество по 8 критериям и сохраняет в базу знаний для последующего поиска.

## Стек

- **LangGraph** — граф состояний агента
- **ChromaDB** — векторное хранилище для RAG
- **OpenAI API** — генерация и эмбеддинги (gpt-4o-mini, text-embedding-3-small)
- **aiogram 3** — Telegram-бот
- **Python 3.14**

## Быстрый старт

```bash
# 1. Активировать venv (находится в родительском каталоге)
source ../.venv/bin/activate

# 2. Скопировать и заполнить переменные окружения
cp .env.example .env
# Вставить OPENAI_API_KEY и TELEGRAM_BOT_TOKEN

# 3. Сгенерировать синтетическую базу PRD
python generate_synthetic.py --count 50

# 4. Проиндексировать базу
python search.py --index

# 5. Запустить бота
python bot.py
```

## Команды бота

| Команда | Описание |
|---|---|
| `/new <описание>` | Запустить генерацию PRD |
| `/skip` | Пропустить уточняющие вопросы |
| `/search <запрос>` | Найти PRD в базе |
| `/critique <номер>` | Оценить PRD из результатов поиска |

После `/critique` можно задавать follow-up вопросы прямо в чат:
- `почему метрики получили 1?` — объяснение по критерию (по ключевому слову)
- `расскажи про критерий 3` — по номеру критерия
- любой другой текст — объяснения по всем слабым критериям

## Флоу агента

```
PM пишет /new <описание фичи>
    │
    ▼
Агент ищет похожие PRD в RAG (ChromaDB)
    │
    ▼
Агент задаёт 3–5 уточняющих вопросов
    ├── PM отвечает → агент учитывает ответы
    └── PM пишет /skip → PRD с пометкой "требует доработки"
    │
    ▼
Агент генерирует PRD по шаблону
    │
    ▼
Критик оценивает PRD по 8 критериям (макс 16 баллов, порог 11)
    ├── Прошёл → сохранение и отправка
    └── Не прошёл и есть прогресс → повторная генерация с замечаниями
    │
    ▼
PRD сохраняется в data/prd_output/ и индексируется в ChromaDB
Отправляется .md файл в Telegram с оценкой качества
```

## Критик PRD

Пакет `critique/` оценивает документ по 8 критериям (по 0–2 балла каждый):

| Критерий | Что проверяет |
|---|---|
| Метрики измеримы | Есть конкретные цифры и сроки, а не «улучшить» |
| Сегмент пользователей | Целевая аудитория описана конкретно |
| Функц. требования | Что система должна уметь |
| Out of scope | Что явно не входит в эту версию |
| Открытые вопросы | Нерешённые вопросы зафиксированы |
| Отсутствие воды | Нет расплывчатых формулировок |
| JTBD описан | Есть формулировка: «Когда… я хочу… чтобы…» |
| Бизнес-метрика | Есть ключевая метрика с текущим и целевым значением |

Подробнее — в [`critique/CLAUDE.md`](critique/CLAUDE.md).
