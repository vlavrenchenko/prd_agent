# Ручные тесты: объяснения по критериям после /critique

Фича: после команды `/critique` пользователь может задать follow-up вопрос
и получить объяснение почему LLM поставил ту или иную оценку.

---

## Предусловия

- Бот запущен: `python bot.py`
- В базе есть хотя бы один PRD (или сгенерирован через `/new`)
- Переменная `TELEGRAM_BOT_TOKEN` задана в `.env`

---

## TC-01 Объяснение по ключевому слову критерия

**Шаги:**
1. `/search онбординг`
2. `/critique 1`
3. Написать: `почему метрики получили 0?`

**Ожидаемый результат:**
- Шаг 2: показана таблица баллов по 8 критериям (эмодзи ✅/⚠️/❌, формат `X/2`)
- Шаг 3: ответ содержит только объяснение по критерию `Метрики измеримы`, балл и текст LLM

---

## TC-02 Объяснение по номеру критерия

**Шаги:**
1. `/search онбординг`
2. `/critique 1`
3. Написать: `расскажи про критерий 3`

**Ожидаемый результат:**
- Шаг 3: ответ содержит объяснение по критерию №3 (`Функц. требования`)

---

## TC-03 Объяснение по ключевому слову JTBD

**Шаги:**
1. `/search онбординг`
2. `/critique 1`
3. Написать: `что с jtbd?`

**Ожидаемый результат:**
- Шаг 3: объяснение по критерию `JTBD описан`

---

## TC-04 Fallback — нет совпадения с критерием

**Шаги:**
1. `/search онбординг`
2. `/critique 1`
3. Написать: `расскажи подробнее`

**Ожидаемый результат:**
- Шаг 3: показаны объяснения по **всем** критериям с оценкой < 2 (не одному)

---

## TC-05 Все критерии максимальные

**Шаги:**
1. Найти или сгенерировать PRD с высоким качеством (оценка ≥ 11/16)
2. `/critique <номер>`
3. Дождаться ответа с оценкой 16/16
4. Написать любой текст, например: `а подробнее?`

**Ожидаемый результат:**
- Шаг 4: `✅ Все критерии получили максимальный балл.` (без перечисления критериев)

---

## TC-06 Сообщение без активного critique_cache

**Шаги:**
1. Написать любой текст **без** предварительного `/critique`

**Ожидаемый результат:**
- Ответ: предложение начать с `/new` или `/search` (стандартный хэлп)
- Никаких объяснений по критериям не выдаётся

---

## TC-07 Активная сессия генерации имеет приоритет над critique_cache

**Шаги:**
1. `/search онбординг`
2. `/critique 1` (заполняется critique_cache)
3. `/new виш-лист` (создаётся active_session, бот задаёт уточняющие вопросы)
4. Ответить на вопросы агента обычным текстом

**Ожидаемый результат:**
- Шаг 4: бот воспринимает текст как ответ агенту и продолжает генерацию PRD
- Никакого объяснения по критериям не выдаётся

---

## TC-08 Номер критерия вне диапазона 1–8

**Шаги:**
1. `/search онбординг`
2. `/critique 1`
3. Написать: `объясни критерий 9`

**Ожидаемый результат:**
- Шаг 3: fallback — показаны объяснения по всем слабым критериям
  (9 вне диапазона, число игнорируется)

---

## TC-09 Хэлп упоминает follow-up вопросы

**Шаги:**
1. `/start`

**Ожидаемый результат:**
- Сообщение содержит раздел с примерами follow-up вопросов после `/critique`
  (например: `почему метрики получили 1?`, `расскажи про критерий 3`)
